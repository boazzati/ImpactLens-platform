CORS Issue Solution

The problem: Browser security policies block frontend requests from Vercel to Heroku

Solution: Implement a Vercel Proxy API Route

Create a serverless function on Vercel that acts as a proxy between your frontend and Heroku backend.

// api/proxy.js in your Vercel frontend project export default async function handler(req, res) { const { method, body } = req; try { const response = await fetch('https://impactlens-platform-20d6698d163f.herokuapp.com/api/test-openai', { method, headers: { 'Content-Type': 'application/json', }, body: JSON.stringify(body) }); const data = await response.json(); res.status(200).json(data); } catch (error) { res.status(500).json({ error: error.message }); } }

Then update your frontend to call the proxy endpoint instead of the Heroku backend directly:

// Updated fetch call in your React component const response = await fetch('/api/proxy', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify(requestData) });
This solution works because the frontend and proxy are on the same domain (Vercel), avoiding CORS issues.
